## 0. Two findings that reframe the prompt

1. **Nothing to port from V1.** A grep of the whole `reference/v1/` tree found
   **no** tax logic (no `medicareLevy` / brackets / `estimatedRefund` /
   `isTaxDeductible`) and **no** file-export logic. `reportService.ts` only
   builds report *data objects*; it never emits CSV/PDF. Both the AU
   tax-estimate math **and** the export are entirely **net-new** for V2.
2. **There is a design contract for the tax view.**
   `design_handoff_upshot/build/surfaces.jsx` → `TaxTime()` (and
   `screens/expansion.jsx`) is the visual spec: under the **Analyse** room
   (`look` rail), three stat cards — **Deductible total**, **Est. deduction
   benefit** ("at … marginal rate"), **Est. refund position** ("estimate · not
   advice") — over a **Deductible-by-category** table. This confirms
   `/analyse/tax` and its layout.

Confirmed AU **FY2025-26 resident brackets** (ATO): nil ≤ $18,200 · 16% to
$45,000 · 30% to $135,000 · 37% to $190,000 · 45% above. Medicare levy 2%. The
16% band drops to **15% from 1 Jul 2026 (FY2026-27)** — so brackets are a
per-FY table, not a single hardcoded set.

## 2. New core modules (pure, integer-cents, never import `@upshot/db`)

### `packages/core/src/tax/brackets.ts`
- Resident bracket table keyed by FY-ending year. FY2026 (= 2025-26) seeded;
  table shape lets FY2027 (15% band) be added later by data, not code change.
- `incomeTaxCents(taxableIncomeCents: number, fy: number): number` — progressive
  bracket math, integer cents.
- `marginalRatePct(taxableIncomeCents: number, fy: number): number` — the
  marginal **rate** (float ratio, e.g. `0.30`) at that income (rates/percentages
  may be float; cents may not).

### `packages/core/src/tax/estimate.ts`
`buildTaxEstimate(input): TaxEstimate` where
`input = { grossIncomeCents, paygWithheldCents, deductibles: {category, cents, count}[], medicareLevyApplies, fy }`:

- `totalDeductibleCents = Σ deductibles.cents`
- `taxableIncomeCents = max(0, grossIncomeCents − totalDeductibleCents)`
- `incomeTaxCents = incomeTaxCents(taxableIncome, fy)`
- `medicareLevyCents = medicareLevyApplies ? round(taxableIncome × 0.02) : 0`
  — **flat 2%**, low-income reduction omitted (documented simplification).
- `litoCents` — Low Income Tax Offset, **non-refundable**, offsets income tax
  only (not Medicare). Exact phase-out figures verified at plan time
  (2025-26 standard: max $700 ≤ $37,500; −5c/$ to $45,000; −1.5c/$ to $66,667).
- `liabilityCents = max(0, incomeTaxCents − litoCents) + medicareLevyCents`
- `refundPositionCents = paygWithheldCents − liabilityCents` (+ = refund,
  − = owing)
- `marginalRatePct` (for the benefit-card label)
- `deductionBenefitCents = liability(grossIncome, no deductions) − liability(with deductions)`
  — the **true** saving (correctly handles bracket-straddle), not a flat
  marginal multiply.
- `byCategory[]` (category, cents, count), `totalDeductibleCents`,
  `flaggedCount`.

### `packages/core/src/export/csv.ts`
- Generic `toCsv(headers: string[], rows: string[][]): string` — RFC-4180
  escaping (wrap in quotes when a cell holds `,` `"` or newline; double interior
  quotes); `\r\n` line endings.
- Per-report serialisers: `taxEstimateToCsv`, `yearlyReportToCsv`,
  `monthlyReportToCsv`. Cents → dollar strings formatted **at the edge** via the
  existing money formatter — no `parseFloat`.

All four modules unit-tested; money math property-tested (no cent lost/invented).

## 4. Web surfaces (mirror the forecast pattern: page → `data.ts` → view)

- **`app/(app)/analyse/tax/page.tsx`** (`export const dynamic = "force-dynamic"`)
  → `loadTaxData(db, { now })` in `./data.ts`:
  - FY window from `financialYearStartMonth` (reuse the windowing already in
    `packages/core/src/reports/yearly.ts`).
  - Sum **settled** `isTaxDeductible` transactions inside the window, grouped by
    `taxDeductionCategory` (fall back to the Up category name when null);
    counts per group.
  - Load settings (gross, withheld, `medicareLevyApplies`, FY start) and call
    `buildTaxEstimate`.
- **`components/analyse/tax-view.tsx`** — three stat cards (Deductible total ·
  Est. deduction benefit · Est. refund position) + Deductible-by-category table,
  per the mock. AUS English ("financial year", "deductible"). Disclaimers
  ("estimate · not advice"; the exclusion list). **Export CSV** + **Print / Save
  as PDF** buttons. When `gross == 0 && withheld == 0`: still show deductible
  total + by-category; replace the two refund cards with a prompt linking to
  Settings → Tax.
- Add `{ label: "Tax", href: "/analyse/tax" }` to `ANALYSE_SECTIONS` in
  `apps/web/components/analyse/analyse-nav-rail.tsx`.

### Export wiring
- `apps/web/server-actions/export-core.ts` — pure, db-injected, tested:
  `buildTaxCsv(db, {now})`, `buildYearlyCsv(db, {fy})`, `buildMonthlyCsv(db, {month})`,
  each returning `{ filename, csv }`.
- `apps/web/server-actions/export.ts` — thin `"use server"` wrapper
  (`action()`-wrapped) calling the core fns. **Only async fns + `export type {…}
  from "…"`** (no bare `export type`).
- Client: an `ExportButton` that calls the action and triggers a Blob download
  (`a[download]`). Read-only path, but a real invoked path (satisfies the
  e2e-write-path rule).
- **Print/PDF:** a "Print / Save as PDF" button calling `window.print()`, plus
  `@media print` rules that hide the rail, top bar, sub-nav, and action buttons,
  leaving a clean report. No dependency.
- FY + Monthly report views (under `/analyse`) gain the same CSV + print buttons.
