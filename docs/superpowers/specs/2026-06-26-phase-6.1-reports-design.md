folded-in enhancements (month-over-month deltas, no-spend streaks, auto-insights).

## 4. Architecture & layering

```
packages/core/src/reports/      ← pure report math (TDD, integer cents, DB-free)
        ↓ (typed cents DTOs)
packages/ui/src/finance/        ← chart primitives (visx + bespoke SVG)
        ↓
apps/web/app/(app)/analyse/     ← room layout + Reports + Analytics surfaces
```

- **Core is pure and DB-free**: functions take already-loaded rows (transactions,
  categories, savers, debts, valuations) and return cents DTOs — matching the existing
  `budget`/`networth`/`payoff` module pattern. No Drizzle imports in `core/reports`.
- **Web server components** load rows via repos and call core; client components render charts.
- **No float anywhere.** Every V1 `number` money field becomes an integer `*Cents` field.

## 5. Core report math — `packages/core/src/reports/`

Ported from `reference/v1/backend-src/services/`, reshaped to cents + pure functions.

| Module | Ports from V1 | Produces |
|---|---|---|
| `salary-periods.ts` | `reportService.getSalaryPeriods` (+ calendar fallback) | `SalaryPeriod[]` from `isSalary` txns. **Pure** — detection stays in the rules engine (MARK_SALARY); this only derives windows. |
| `monthly.ts` | `generateMonthlyReport` | `MonthlyReport` (incomeCents/expenseCents/netCents, categoryBreakdown, envelopePerformance, debtPaidCents, savingsRate, insights) |
| `yearly.ts` | `generateYearlyReport` + `generateFinancialYearReport` | `YearlyReport` (calendar year + AU FY, monthlyBreakdown, prev-year comparison) |
| `category-breakdown.ts` | `buildCategoryBreakdown` | category shares (cents + % of total), sorted desc |
| `cashflow.ts` | `/reports/cashflow` (historical only) | income-vs-expense series over a period |
| `tag-summary.ts` | `getTagSummary` | per-tag totals (cents + count) |
| `behavioural.ts` | `insightsService.getBehavioralInsights` + `getSpendingInsights` | behavioural patterns, spending insights, **no-spend streaks**, **heatmap daily-spend bins** |
| `budget-health.ts` | `budgetAnalysis.calculateBudgetHealth` (not yet in V2 core) | `BudgetHealthScore` |
| `insights.ts` | `Insight` generators across the above | warning/positive/info insight DTOs |

**Reused from existing V2 core (Phase 4) — not re-ported:** `analyseSaver` /
`analyseAllSavers` (envelope/saver alignment), `analyseEmergencyFund` + `ReadinessGauge`
(emergency-fund readiness), `goalConfidence`.

### Salary-period derivation (the V2-new piece)
Order `isSalary` transactions ascending by date (`settledAt ?? createdAt`); each period =
`[salaryDate, nextSalaryDate − 1 day]`, the last open-ended through "now"; return
most-recent-first with reassigned indices (index 0 = current period). If no salary txns
exist, fall back to calendar-month periods. Label e.g. `"15 Jan – 14 Feb 2026"`.

### Data inputs (loaded by web, passed to core)
transactions (with `amountCents`, `isSalary`, `isTransfer`, `categoryId`, `settledAt`,
`createdAt`, tags), categories (+ parent), savers/envelopes + allocations, debts + derived
debt payments (Phase 5 `effectiveDebtPaymentCents`), asset valuations / monthly snapshots
(for NetWorthTrend). All present in V2 as of Phases 2/4/5.

## 11. Constraints & scars (must hold)

- Integer cents only; never `parseFloat`. `Money`/`MoneyInput` take integer cents.
- All colours via `tokens.css` `var(--…)` by name — no hex. `color-mix(in oklch, …)` allowed.
- `"use server"` modules export **only** async functions + `export type {X} from "..."`
  (a bare `export type {X}` crashes every action).
- Run `pnpm --filter @upshot/web typecheck` independently after **every** web task (vitest hides type errors here).
- `exactOptionalPropertyTypes` is **not** enabled — conditional-spread idiom is house style; typecheck is the arbiter.
- e2e is one test on one context — extend `auth.spec.ts`, invoke real paths, keep final `expect(pageErrors).toEqual([])`.
- SDD ledger lives at `.superpowers/sdd/progress.md` (gitignored).
